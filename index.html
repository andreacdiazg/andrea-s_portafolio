GitHub Portfolio Setup Guide

Learn exactly where to add your HTML code and publish your portfolio

Where to Add Your HTML Code on GitHub

1

Create a New File

In your GitHub repository, click the "Add file" button and select "Create new file".

username.github.io

No files yet. Click "Add file" to add your HTML file.

Your empty GitHub repository
2

Name Your File

Name your file index.html - this is the default file that GitHub Pages will look for.

username.github.io / index.html
Name your file...
index.html
Naming your file index.html
3

Paste Your HTML Code

Paste your complete HTML code into the editor. Make sure to include the entire document from <!DOCTYPE html> to </html>.

// Paste your entire HTML code here
<!DOCTYPE html>
<html lang="en">
<head>
   <meta charset="UTF-8">
   <title>My Portfolio</title>
</head>
<body>
   <h1>Welcome to My Portfolio</h1>
</body>
</html>
4

Commit Your Changes

Scroll to the bottom of the page, add a commit message, and click "Commit new file".

Commit new file
Commit directly to the main branch
Committing your new HTML file
5

Enable GitHub Pages

Go to your repository Settings, then to the Pages section. Select the main branch and click Save.

GitHub Pages

Enabling GitHub Pages in repository settings
6

View Your Live Portfolio

After a few minutes, your portfolio will be live at https://username.github.io (replace "username" with your GitHub username).

Success! Your portfolio is now live. It may take a few minutes for the site to be available after your first commit.

Pro Tip

If you already have an HTML file, you can also upload it directly by clicking "Add file" → "Upload files" and dragging your HTML file to the repository.

Frequently Asked Questions

Q: Why does my file need to be named index.html?

A: index.html is the default file that web servers look for when a URL doesn't specify a filename. This makes your portfolio accessible at a clean URL without needing to include the filename.

Q: My site isn't showing up. What should I do?

A: First, check that your repository is set to public. Then verify that GitHub Pages is enabled for the main branch. It can take up to 10 minutes for your site to be available after the first deployment.

Q: Can I use a custom domain?

A: Yes! In your repository Settings → Pages, you can add a custom domain. You'll also need to configure your DNS settings with your domain registrar.

Q: How do I update my portfolio later?

A: You can edit your index.html file directly on GitHub by clicking on the file and then the pencil icon, or upload a new version of your HTML file.
